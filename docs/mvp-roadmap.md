## Phase 1

Goal: establish a trustworthy data model and stage contracts.

- define `SourceRecord`
- define `HypothesisCard`
- define scoring and promotion criteria
- define pipeline stages from acquisition to arbiter

## Phase 2

Goal: bootstrap a narrow evidence flow for 3 domains.

- load curated source notes
- normalize source metadata
- extract mechanisms, methods, limitations, and unresolved phenomena
- assemble cross-domain candidate pairs

## Phase 3

Goal: generate and filter candidate hypotheses.

- produce candidate hypothesis cards
- run skeptic objections
- score and rank candidates
- cluster near-duplicates

## Phase 4

Goal: move from ideas to validation.

- attach Python-first test plans
- define data and simulation routes
- run baseline computational checks for the strongest cases
- promote candidates into a `top-5` board

## Phase 5

Goal: publication-grade packaging.

- prepare evidence dossiers
- record prior-art review gaps
- package reproducible notebooks or scripts
- prepare short scientific briefs for external review
