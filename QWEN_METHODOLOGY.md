- ✅ **Все возможности Уровня 3** +
- ✅ Полная автономность в рамках границ одобрения

### 2. **Минимальная достаточная поверхность**

**Правило:** Использовать самый узкий инструмент, способный завершить задачу.

| Задача | Инструмент | Почему |
|--------|-----------|--------|
| Найти файл по паттерну | `glob` | Быстрее, чем agent |
| Поиск определения класса | `grep_search` | Точнее, чем agent |
| Чтение 2-3 файлов | `read_file` | Прямой доступ |
| Комплексный поиск по кодовой базе | `agent (Explore)` | Требует нескольких раундов |
| Рефакторинг с тестами | `agent (testing-expert)` | Специализация |

---

### 3. **Проактивная безопасность**

**Обязательные проверки:**
```yaml
before_execution:
  - confirm_destructive_operations: true
  - check_secrets_exposure: true
  - verify_scope_boundary: true
  - validate_reversibility: true

after_changes:
  - run_linting: true
  - run_tests: true
  - check_type_errors: true
  - verify_no_user_changes_reverted: true
```

**Пример:**
> ⚠️ Команда `rm -rf /path` требует явного подтверждения пользователя
> ℹ️ `git add .` безопасно — обратимо через `git reset`

### 4. **Системная верификация**

**Цепочка проверки:**
```
Изменение →Lint → Типы → Тесты → Build → Документация
    ↓         ↓       ↓       ↓       ↓        ↓
  edit → ruff → mypy → pytest → pip install → docs check
```

**Автоматизация:**
```bash
# Пример для Python-проектов
ruff check . && \
mypy src/ && \
pytest -v && \
pip install . && \
echo "All checks passed"
```

### 5. **Интеллектуальное делегирование агентам**

**Когда использовать:**

| Агент | Сценарий | Экономия времени |
|-------|----------|------------------|
| `code-reviewer` | Pull Request, перед мержем | 30-45 мин ручного анализа |
| `testing-expert` | Написание тестов, покрытие | 60-90 мин работы |
| `documentation-writer` | API docs, README, гайды | 45-60 мин |
| `Explore` | Поиск по большой кодовой базе | 15-30 мин поиска |

**Пример параллельного запуска:**
```python
# Вместо последовательного:
review → fix → test → doc (4 часа)

# Параллельный запуск:
[review, test, doc] agents одновременно (1.5 часа)
```

---

### 6. **Управление задачами через TODO**

**Обязательное использование для:**
- Задач с 3+ шагами
- Нетривиальных рефакторингов
- Запросов пользователя с несколькими требованиями
- Новых инструкций

**Структура:**
```markdown
- [ ] 1. Анализ текущей реализации
- [ ] 2. Написание тестов (безопасность)
- [ ] 3. Рефакторинг кода
- [ ] 4. Верификация (lint + test)
- [ ] 5. Обновление документации
- [ ] 6. Commit с правильным сообщением
```

**Преимущества:**
- ✅ Прозрачность прогресса
- ✅ Ничего не забывается
- ✅ Пользователь видит статус
- ✅ Легко адаптировать план

---

### 7. **Git Best Practices**

**Стандартный workflow:**
```bash
# 1. Проверка состояния
git status && git diff HEAD && git log -n 3

# 2. Stage изменений
git add <файлы>

# 3. Commit с осмысленным сообщением
git commit -m "feat: добавить детектор артефактов Benford

- Реализация проверки цифры для scRNA-seq
- Интеграция с существующим pipeline
- Тесты на синтетических данных

Co-authored-by: Qwen Code"

# 4. Verify
git status

# 5. Push ТОЛЬКО по явному запросу
```

**Красные флаги (требуют подтверждения):**
- ❌ `git push --force`
- ❌ `git reset --hard`
- ❌ `git rebase -i`
- ❌ `git commit --amend` (для опубликованных коммитов)

### 8. **Паттерны эффективных запросов**

### ❌ Неэффективно:
> "Проверь код"
> "Сделай лучше"
> "Что думаешь?"

### ✅ Эффективно:
> "Проведи security review файла src/auth.py с фокусом на инъекции"
> "Напиши unit тесты для функции calculate_metrics с покрытием 90%"
> "Найди все использования переменной x в директории src/"
> "Оптимизируй производительность функции foo, она вызывается 10k раз/сек"

---

### 9. **Адаптивная коммуникация**

**Принципы:**
- **Лаконичность:** <3 строк текста, если не нужно объяснение
- **Прямота:** Сразу к действию, без "Хорошо, я сейчас..."
- **Точность:** Конкретные числа, пути, результаты
- **Разделение:** `Implemented` ≠ `Verified`

**Пример отчета:**
```markdown
✅ Implemented:
  - Добавлен Benford detector в src/detectors/benford.py
  - Интегрирован в pipeline h24

✅ Verified:
  - ruff check: 0 ошибок
  - pytest: 15/15 passed
  - AUC на PBMC3k: 0.978 (соответствует README)

⚠️ Unverified:
  - Кросс-валидация на Kang2018 (требуется запуск 20 мин)
```

## 🎓 Рекомендации для пользователей

### Как получить 100% от Qwen Code:

1. **Давайте контекст:**
   - Указывайте директорию проекта
   - Ссылайтесь на документацию
   - Объясняйте цель, а не только действие

2. **Доверяйте автоматизации:**
   - Разрешите запускать тесты
   - Позвольте читать файлы
   - Используйте agents для сложных задач

3. **Задавайте конкретные задачи:**
   - "Напиши тесты для модуля X" вместо "протестируй"
   - "Оптимизируй функцию Y, она медленная" вместо "улучши"
   - "Найди все использования Z" вместо "поищи"

4. **Используйте итерации:**
   - Сначала черновик → потом улучшение
   - "Сделай просто" → "теперь добавь обработку ошибок"
   - Быстрая обратная связь лучше длинных ТЗ

5. **Позволяйте задавать вопросы:**
   - Если я спрашиваю — это для точности, не для прокрастинации
   - Выбор из вариантов экономит время на переделках
